这是一个Rust语法练手项目，目标是学习Rust的语法和并发模型。

以下是这四个项目具体的目标定义（MVP - 最小可行性产品）以及最终运行时的预期效果。其中，每一个项目都有自己的目标和预期效果，代码部分有些琐碎和Rust语法关系不大的部分可以直接用LLM工具生成， 测试用例部分也用LLM工具生成， 只需要关注核心功能的实现，以此快速完成一个MVP项目实现Rust开发能力的提升。

注意：LLM工具生成的代码需要有注释，LLM也要生成测试用例，所有项目共享一个Rust工程，但是每一个项目一个目录，有自己的main函数， 需要手工补全的部分记得标记TODO，不要生成重点手写区的代码！

1. 命令行 HTTP 压测工具 (CLI Load Tester)
核心目标：
做一个体验良好的终端工具，不仅能用，还要好用（有进度反馈、有格式化输出）。彻底搞懂 tokio 是如何调度海量并发请求的。
最终做成的效果：
启动与执行： 在终端中输入编译好的二进制命令，例如 ./my_hey -c 50 -n 1000 http://localhost:8080（代表 50 个并发，共发送 1000 个请求）。
实时反馈： 终端屏幕上会出现一个动态更新的进度条，显示当前已经完成了多少个请求，让用户知道程序没有卡死。
结果汇总： 压测结束后，终端会打印出一份整洁的报告，包含：
成功和失败的请求数量。
总耗时、平均每秒请求数 (QPS/RPS)。
延迟分布：最快耗时、最慢耗时、平均耗时，最好能算出一个简单的 P90 或 P99 延迟（把所有响应时间排序，取第 90% 或 99% 位置的值）。
✍️ 重点手写区（锻炼 Rust 核心肌肉）：

并发任务调度 (tokio::spawn)： 手写如何创建数千个并发任务。这需要你深入理解闭包（Closure）的所有权转移（move 关键字），以及 Send 和 Sync 这两个并发 Trait。

数据聚合与通道通信 (mpsc channel)： 手写一个接收器循环，聚合各个 HTTP 请求的结果（耗时、状态码）。体会如何在多任务间安全地传递消息。

错误传播 (? 操作符)： 优雅地处理网络超时、DNS 解析失败等错误，不写满屏的 if err != nil。

🛠️ 工具/库代劳区（直接调用，不要造轮子）：

命令行参数解析： 使用 clap 库搭配 #[derive(Parser)] 宏。只需定义一个 Struct，宏会自动帮你生成帮助文档、参数校验和解析代码。

HTTP 协议封装： 使用 reqwest 库发送请求，不要自己去写 TCP Socket 和 HTTP 报文解析。

进度条展示： 接入 indicatif 库，直接调用 API 渲染漂亮的终端进度条。

2. 线程安全的内存级 KV 数据库 (Concurrent Key-Value Store)
核心目标：
写一个长期运行的守护进程（Daemon）。不依赖任何 HTTP 框架，直接在 TCP 层面处理裸字节流。在这个过程中，学会用 Arc<RwLock<HashMap>> 保护数据。
最终做成的效果：
服务端运行： 在终端运行服务端 ./kv_server，程序挂起并在控制台打印 Listening on 127.0.0.1:6379...。
多客户端交互： 打开两到三个新的终端窗口，使用系统自带的 nc (netcat) 命令连接服务端：nc 127.0.0.1 6379。
并发操作：
在客户端 A 输入：SET name Rust，服务端返回 +OK。
在客户端 B 输入：GET name，服务端立刻返回 Rust。
在客户端 C 输入非法指令或突然断开连接（Ctrl+C），服务端控制台会打印一条警告日志，但绝对不会崩溃，依然继续服务 A 和 B。

✍️ 重点手写区（锻炼 Rust 核心肌肉）：

共享状态的锁机制 (Arc<RwLock<T>>)： 手写底层存储引擎。这是重头戏，你需要设计如何将 HashMap 安全地暴露给多个 TCP 连接。在这个过程中，借用检查器会严厉地教你做人。

自定义二进制协议解析： 手动实现一个将字节流（&[u8]）切割并转化为 Command 枚举（如 Get(String), Set(String, String)）的解析器。这是练习 Rust 强大模式匹配（Pattern Matching）和切片（Slice）生命周期的绝佳场地。

🛠️ 工具/库代劳区（直接调用，不要造轮子）：

底层异步 I/O： 直接使用 tokio::net::TcpListener 处理底层的 TCP 握手和字节流读写。

日志系统： 使用 tracing 库打印带有上下文的结构化日志，代替 println!。

3. WebAssembly Markdown 解析器 (WASM Interop Module)
核心目标：
体会 Rust 的跨平台能力，用 Rust 写纯逻辑，用 JavaScript 写 UI，两者通过 WASM 完美结合。掌握 Rust 中最核心的 String、&str 和迭代器的高级用法。
最终做成的效果：
产物编译： 运行打包命令后，Rust 代码会生成一个 pkg 文件夹，里面包含一个极其小巧的 .wasm 文件和一个用来做桥接的 .js 文件。
浏览器运行： 写一个极简的 index.html（不需要复杂的打包工具，直接引入生成的 JS 文件即可）。页面左边是一个 <textarea> 输入框，右边是一个 <div> 预览区。
极速渲染： 当你在左边的输入框敲下 # Hello Rust 时，JavaScript 会捕获键盘事件，调用 WASM 中的 Rust 函数，Rust 瞬间完成解析并返回 HTML 字符串 <h1>Hello Rust</h1>，右边的预览区实时更新。整个过程流畅无卡顿。


✍️ 重点手写区（锻炼 Rust 核心肌肉）：

抽象语法树 (AST) 设计： 用 Rust 的 Enum 定义 Markdown 的节点类型（标题、列表、加粗等）。Rust 的枚举本质上是代数数据类型（ADT），比 TS 的联合类型更底层且高效。

状态机解析逻辑： 手写遍历字符串字符的逻辑，练习使用迭代器 (Iterator) 的高级组合操作（如 peekable, take_while），并处理复杂的字符串所有权（String vs &str）。

🛠️ 工具/库代劳区（直接调用，不要造轮子）：

WASM 编译与 JS 胶水代码： 绝对不要手动去写内存导入导出的 FFI 接口。直接在 Rust 函数上标记 #[wasm_bindgen] 宏，然后用 wasm-pack build 命令一键生成 .wasm 文件和带有完整类型提示的 .d.ts 文件。
4. 极简 RESTful API 后端 (Web API Service)
核心目标：
搭建一个具备企业级雏形的 Web 服务，体验 Rust 严格的编译期检查是如何保证业务逻辑严谨性的。重点感受框架的依赖注入和错误统一拦截。
最终做成的效果：
数据持久化： 程序启动时，自动连接到本地的 SQLite 数据库文件。如果表不存在，程序会自动执行初始化 SQL 建表。
API 交互 (以 Todo 为例)： 使用 Postman 或 curl 进行测试。
发送 POST /todos，Body 带上 {"title": "学习所有权", "completed": false}，服务端存入数据库并返回 HTTP 201 Created 和生成的 ID。
发送 GET /todos，服务端从数据库查出列表，返回格式化的 JSON 数组。
极致的错误处理： 如果用户发送了恶意的 JSON（比如 completed 字段传了字符串而不是布尔值），或者请求了一个不存在的 ID，框架会自动拦截并返回一致的、自定义格式的 HTTP 400/404 错误 JSON，而不是直接抛出异常导致进程崩溃。

✍️ 重点手写区（锻炼 Rust 核心肌肉）：

业务逻辑层与领域模型： 手写标准的 Rust Struct，并为它们实现各类特征（Traits），例如实现 From 特征来优雅地完成 DTO 到 DO 的实体转换。

全局错误处理： 手写一个自定义的 AppError 枚举，并为它实现 IntoResponse。这样你的路由处理函数就可以直接返回 Result<T, AppError>，框架会自动将错误映射为标准的 HTTP 500/400 JSON 响应。

🛠️ 工具/库代劳区（直接调用，不要造轮子）：

JSON 序列化/反序列化： 使用 serde 库。只需在实体类上加上 #[derive(Serialize, Deserialize)]，它会在编译期生成极速的解析代码（比反射快得多）。

SQL 拼接与数据库映射： 使用 sqlx 库。它的 query! 宏会在编译期连接你的本地数据库，校验你的 SQL 语法和字段类型。如果 SQL 写错了，代码直接无法编译。

路由解析： 使用 axum 框架，它通过强大的类型系统自动处理 HTTP 提取器（Extractors，如 Path、Query、Json Body）。
